AWS Project Summary

Self-Hosted Streaming & Media Management Platform for Deployed Military Personnel

This project will deploy Jellyfin, a self-hosted media streaming platform, along with automated media acquisition and management tools. The system will be distributed across AWS and GCP for redundancy, with Ansible for automation and AWS services for storage, caching, security, and access control

Provision VMs

      • AWS needs a keypair to connect
      • Modified the security group inbound/outbound to accept TCP:22
      • Added the AWS key to "authorized_hosts" in GCP for ease of use

Refining the Ansible Playbook

  1. Encountered package dependency conflicts with curl-minimal on AWS instances.
    • Modified the playbook to remove curl-minimal before installing Docker dependencies.
  2. Adjusted the playbook to use yum exclusively for AWS instances.
    • Simplified logic and removed unnecessary apt checks.
  3. Resolved Docker installation issues by using Amazon Linux Extras repository.
    • Added task to enable the docker extras repository.
    • Simplified Docker installation to use docker package.

Troubleshooting Amazon Linux Extras

  1. Encountered "amazon-linux-extras not found" error on AWS instances.
    • Identified the instance as Amazon Linux 2023.
    • Added a task to install amazon-linux-extras package if missing.
  2. Resolved repository metadata errors and package conflicts.
    • Removed incorrect Docker CE Stable repository.
    • Cleaned yum cache and updated repositories.
    • Installed dnf-plugins-core to ensure repository functionality.

Implementing Sudo Password Variables

  1. Encountered "Missing sudo password" errors during Ansible execution.
    • Implemented Ansible Vault to securely store sudo passwords.
    • Modified playbook to use ansible_sudo_pass from Vault.
    • Used --ask-vault-pass to provide Vault password during execution.
